函数的四种调用模式

May 20, 2016 | js
  1. 函数模式
  2. 方法模式
  3. 构造函数模式
  4. 上下文模式

函数模式

特征: 就是一个简单的函数调用. 函数名的前面没有任何引导内容.

this 的含义: 在 函数中 this 表示全局对象, 在浏览器中是 window

方法模式

特征: 方法一定是依附于一个对象, 将函数赋值给对象的一个属性, 那么就成为了方法.

this 的含义: 这个依附的对象.

构造函数调用模式

上下文调用模型

上下文 就是环境, 就是自定义设置 this 的含义

语法:

  1. 函数名.apply( 对象, [ 参数 ] );
  2. 函数名.call( 对象, 参数 );

定义
1、call/apply定义在Function.prototype上

2、call/apply方法可以实现函数的调用，并且可以指定函数内部的this的值

  • call方法第一个参数为null/undefined,函数内部的this指向window

  • call方法的第一个参数是一个数字、字符串、布尔值，函数内部的this指向他们对应的基本包装类型的对象

  • call方法的第一个参数是一个对象，函数内部的this指向该对象

3、call/apply方法的不同点：给函数传递实参的方式不同其他都相同

应用场景

  • 借用构造函数实现继承
  • 借用Math.max实现计算数组的最大值
  • 借用数组的push方法
  • 借用数组的slice方法
  • 借用Object.prototype.toString检测数据类型
